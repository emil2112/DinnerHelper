## Anti-Patterns Verdict

**LLM assessment**: Passes the AI slop test. Dark forest-green palette is unusual and thematically grounded. No gradient text, no hero-metric cards, no eyebrows on every section. Pantry category labels (10px small-caps) use the eyebrow pattern in a data-label context — defensible but worth watching.

**Deterministic scan**: 1 finding — `overused-font` in `frontend/src/index.css:1`. Inter flagged as saturated and no longer distinctive.

## Overall Impression

Good bones, coherent identity. Biggest opportunity: fix contrast failures (--muted fails WCAG AA throughout) and mobile inaccessibility (sidebar disappears with no alternative).

## What's Working

1. Deep forest green palette earns its identity — doesn't read "tech product"
2. Hero copy ("What sounds good for dinner?") nails the warm, household-familiar tone
3. Layout structure (sidebar + chat + pantry) is well-considered for the use case

## Priority Issues

**[P1] Contrast failure: --muted (#5a7a5a) fails WCAG AA for small text**
- On --bg (#0d1a0d): ~3.72:1
- On --card-bg (#162416): ~3.36:1
- On --input-bg (#0f1f0f): ~3.57:1
All fail the 4.5:1 threshold. Affects: brand tagline, timestamps, category labels, empty states, settings button, placeholder text.
Fix: Shift --muted to ~#7a9e7a (L≈0.30) to clear 4.5:1 against --bg.

**[P1] Mobile: sidebar display:none, chat history inaccessible**
No hamburger menu or drawer. History is completely unreachable on phones.
Fix: Add hamburger in mobile header, slide sidebar as fixed-position drawer overlay.

**[P1] Failed messages have no retry**
Error bubble renders but no retry action. User must retype.
Fix: Store last attempted text, add Retry button on error bubble.

**[P2] Pantry panel always-visible, not collapsible**
260px permanent footprint competes with chat area.
Fix: Default collapsed header, expand on click, persist in localStorage.

**[P2] Header "👤 Our Kitchen" is vestigial**
Person emoji implies user action but does nothing.
Fix: Remove or convert to mobile hamburger trigger.
